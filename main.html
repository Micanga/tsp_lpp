Burma14 - Resolução Heurística¶

A primeira resoluo sugerida ser heurstica.

Nesta heurstica definido que o caminho mais prximo ser o escolhido para constituir a soluo, iterao a iterao.

Para tornar a comparao um pouco mais "justa", iremos realizar a aplicao da heurstica partindo de todos os ns.

In [6]:
# Realizando a aplicao da heurstica para resoluo do problema
println("=====")
s = zeros(N)
for i=1:N
    # copiando a matriz de distancias
    dist_mat_h = zeros(N,N)
    for i=1:N
        for j=1:N
            dist_mat_h[i,j] = dist_mat[i,j]
        end
    end
    
    # procurando pela soluo
    cur_node = i
    visits = union(cur_node)
    while N != length(visits)
        # procurando o menor distncia
        min_dist = 99999
        for j=1:N
            if min_dist > dist_mat_h[cur_node,j] && dist_mat_h[cur_node,j] != 0
                min_dist = dist_mat_h[cur_node,j]
            end
        end
        
        # procurando o proximo n
        dest = -1
        for j=1:N
            if min_dist == dist_mat_h[cur_node,j]
                dest = j
                break
            end
        end
        
        # atualizando os valores na matriz e a soluo
        s[i] = s[i] + dist_mat_h[cur_node,dest]
        dist_mat_h[cur_node,dest] = 99999
        dist_mat_h[dest,cur_node] = 99999
        
        # partindo para o proximo no
        cur_node = dest
        visits = union(visits,cur_node)
    end
    s[i] = s[i] + dist_mat[cur_node,i]
    println("N de partida: ",i," - Distncia: ",s[i])
end 
min_h = minimum(s[1:N])
println("=====")
println("Melhor distncia: ", min_h)

Burma14 - Resolução por MTZ¶

Para garantir uma melhor soluo ao problema, proposto agora a implementao do mtodo de programao inteira mista MTZ.

A soluo apresentada a seguir.

In [7]:
# Realizando a otimizao inteira mista - MTZ
m = Model(with_optimizer(Cbc.Optimizer, logLevel=1))

@variable(m, x[1:N,1:N], Bin)
@variable(m, u[1:N], Int)
@objective(m, Min, sum(x[i,j]*dist_mat[i,j] for i=1:N,j=1:N))
for i=1:N 
    @constraint(m, x[i,i] == 0)
    @constraint(m, sum(x[i,1:N]) == 1)
end
for j=1:N
    @constraint(m, sum(x[1:N,j]) == 1)
end
for f=1:N, t=1:N
    @constraint(m, x[f,t]+x[t,f] <= 1)
end
for i = 2:N
    @constraint(m, u[i] >= 2)
end
for i = 2:N, j = 2:N
    if i != j
        @constraint(m, u[i] - u[j] + N*x[i,j] <= N-1)
    end 
end
redirect_stdout((()->optimize!(m)),open("./null", "w"))
min_mtz = JuMP.objective_value(m)
println("Resultado da funo objetivo: ", min_mtz)

4. Conclusão Geral¶

A resoluo de problemas de otimizao via o mtodo MTZ apresenta um resultado timo em termos de soluo ao um problema modelado a troco de uma penalizao quanto ao tempo necessrio para se encontrar est soluo.

Quando se pretende aplicar mtodos de otimizao, necessrio, acima de tudo, compreender e estudar o problema alvo tentando minimizar as penalizaes relacionadas ao tempo gasto para se melhorar o processo.

Solues heursticas mesmo no apresentando a soluo tima, podem trazer bons resultados, em tempo hbil e com uma porcentagem de perda aceitvel dentro do contexto de aplicao.

Por fim, possvel notar claramente as vantagens e desvantagens associadas a estas modelagens, cabendo ao profissional decidir qual a melhor abordagem a se tomar dentro de seu contexto de atuao.
